      the streaming api of twitter

      Post on Jul 5 2014

      we are doing some research about people’s attitude to Confucius Institute. we need crawl lots of tweets which contains Confucius Institute from twiter. twitter provide streaming api for this purpose. in this article, we will introduce how to use streaming api to crawl tweets which you want.

      there are three streaming endpoints provided : public streams, user streams, site streams.
      in this sence, we choose public streams. there are there endpoints below public streams : POST statuses/filter, GET statuses/sample, GET statuses/firehose. we choose POST statuses/filter, because we want to crawl the tweets which contains keyword.

      the request send to twitter must be authorized according to the OAuth specification. the process of authorizing is a trouble in my try. so, we will introduce this process in detail.

      the first thing of authorizing is collect parameters. there are seven parameters need to be colleted.

      • oauth_consumer_key : this is the API key of your application. before you get it, you should sign up and create a application in twitter.

      • oauthnonce : this is a unique token your application should generate for each unique request.the value is generated by base64 encoding 32 bytes of random data, and stripping out all non-word characters. in python, we can make it use follow codes:


        os.urandom(32) generate 32 bytes random string
        base64.b64encode(str) encode the str in base64
        `re.sub(‘[\W        ]’, ‘’, str)`replace the non-word characters in str with ‘’
        ‘[\W_]’is the regex stands the non-word characters

      • oauth_signature_method : in twitter, the value of this parameter is HMAC-SHA1

      • oauth_timestamp : the timestamp that the requests made. in python, we can make it use follow codes:

      • oauth_token : you can generate a valid token for your account on the API key tab of your application page.

      • oauth_version : the value of this parameter is fixed, that’s is 1.0

      • oauth_signature : this is the most difficult to generate. so we put this in the last

        to generate signature of each request, you should as follows:

        1.collect request method and url
        2.collect parameters

        the parameters include oauth_consumer_key, oauth_nonce, oauth_signature_method, oauth_timestamp, oauth_token, oauth_version and the payload of the request.

        • first, we should percent encode the key-value pairs. in python, you can use this statement to do percent encode : urllib.quote(str2encode, '').
        • second, we should concatenate these key-value pairs in alphabetically sequnence. in python, you can make a dist with these key-value pairs, and sorted it, concatenate it. this str clalled parameter str.
        • third, constuct base string. append the request method and base url delimited by ‘&’, and append the parameter str that have been percent encoded again.
        • forth, get the signing key. the signing key is combined by consumer_secret and token_secret delimited by ‘&’
        • fifth, use hmac-sha1 to generate the signature. in python, do as follows : hmac.new(signkey, basestr, hashlib.sha1).digest(), after that, you should decode the iutput : binascii.b2a_base64(digest)[:-1]

      it’s almost finished. constuct the header string follow the rules, it’s easy, so i dont note it.

      i recommand you send the request with the requests, a network lib for human.
